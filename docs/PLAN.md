# PLAN: SRP Restructure — user/server and user/ui subpackages

## Context

All previous fixes are already applied and published (v0.0.29):
- `// orm:typed_fields` on all DB structs → `User_`, `Session_`, etc. generated
- `ReadAll*` callers already use value return (no `*` dereference needed)
- `models_crud_back.go` wrappers are **redundant** — `ReadAll*` is now generated by `ormc`
- `form.String()` replaces the old `form.RenderHTML()` call

**The only remaining work is restructuring** the package from a flat file layout with build tags
into two clean subpackages with no build tags.

---

## Target Structure

```
user/                  # package user — agnostic types only, no build tags
  models.go            # (unchanged)
  models_orm.go        # (generated, unchanged)
  user.go              # (unchanged)
  generate.go          # (unchanged)

user/server/           # package userserver — backend, no build tags, no wasm imports
  module.go            # Module struct + New() — from user_back.go (drop user_front.go)
  auth.go              # from auth.go
  sessions.go          # session cache + CRUD — from cache.go + sessions.go
  users.go             # from crud.go
  identities.go        # from identities.go
  lan.go               # from lan.go + lan_ips.go
  rbac.go              # from user_rbac_mutations.go
  jwt.go               # from jwt_back.go
  oauth.go             # from oauth.go + google.go + microsoft.go
  middleware.go        # from middleware_back.go
  migrate.go           # from migrate.go
  sql.go               # from sql.go
  api_token.go         # from api_token_back.go
  cache_users.go       # from cache_users.go
  ssr.go               # SSR rendering — from module_*_back.go (all String() methods)

user/ui/               # package userui — frontend, no build tags, no backend imports
  modules.go           # UIModules() + mustForm() — from modules.go
  login.go             # loginModule struct + form + OnMount — from module_login.go + module_login_front.go
  register.go          # from module_register.go + module_register_front.go
  profile.go           # from module_profile.go + module_profile_front.go
  oauth.go             # from module_oauth.go
  lan.go               # from module_lan.go + module_lan_front.go
```

> SSR `String()` methods belong in `user/server/ssr.go` because they are called by the backend
> HTTP handler, not the WASM runtime. `user/ui/` contains ONLY `OnMount()` and form wiring.

## Import changes for downstream packages

`user/server/` and `user/ui/` are sub-packages of the same `go.mod` — no new `go.mod` needed.

| Was | Now |
|---|---|
| `"github.com/tinywasm/user"` → `user.New(...)` | `"github.com/tinywasm/user/server"` → `userserver.New(...)` |
| `"github.com/tinywasm/user"` → `user.UIModules()` | `"github.com/tinywasm/user/ui"` → `userui.UIModules()` |
| `"github.com/tinywasm/user"` → `user.Config`, `user.User` | unchanged — still from root `user` |

Downstream packages to update after restructure: `app`, `goflare`, and any other that calls
`user.New()` or `user.UIModules()`.

---

## Constraints

- No `//go:build` tags in any file after the restructure. Package boundary = build boundary.
- `user/server` must NEVER import `user/ui`.
- `user/ui` must NEVER import `user/server`.
- Both may import root `user` for shared types.
- `models_orm.go` stays at root — generated by `ormc` from `models.go`.
- `user/ui/` must not import `database/sql`, `net/http`, `crypto/*`, `github.com/tinywasm/orm`.
- `user/server/` must not import `syscall/js`, `github.com/tinywasm/dom`, `github.com/tinywasm/form`.
