**Problem**: build tags are the only thing separating backend (DB, HTTP, crypto) from
frontend (DOM, WASM). This causes the WASM binary to be polluted with backend imports at
compile time if a tag is forgotten, forces all readers to context-switch between tags, and
makes it impossible to import only the backend or only the frontend separately.

## Target Structure

### Principle

- **Root (`user/`)**: agnostic types and interfaces only. No `//go:build` tags anywhere at root.
  Contains `models.go`, `user.go`, `generate.go`, `models_orm.go`. No HTTP, no DB, no DOM.
- **`user/server/`**: all `!wasm` code. Package `userserver`. Imports `user` root for types.
  Contains `Module`, `New()`, middleware, sessions, CRUD, migrations, JWT, OAuth, RBAC, etc.
- **`user/ui/`**: all `wasm` / frontend code. Package `userui`. Imports `user` root for types.
  Contains `UIModules()`, module struct definitions, `OnMount()` implementations, form setup.

No `//go:build` tags anywhere. The package boundary IS the build separation:
- WASM binary imports `user` + `user/ui`. Never imports `user/server`.
- Backend binary imports `user` + `user/server`. Never imports `user/ui`.

### Final layout

```
user/
  models.go          # structs, Config, errors, OAuthProvider, AuthMode, SecurityEvent
  user.go            # error vars, type aliases/constants only
  generate.go        # //go:generate ormc
  models_orm.go      # generated by ormc ‚Äî DO NOT EDIT
  go.mod

user/server/         # package userserver ‚Äî backend only, NO wasm imports
  module.go          # Module struct, New()
  auth.go            # password hash/verify
  sessions.go        # session cache + createSession/getSession/deleteSession/cleanExpired
  users.go           # createUser, getUser, getUserByEmail, updateUser, etc.
  identities.go      # createIdentity, getIdentityByProvider, deleteIdentity
  lan.go             # LAN IP helpers (createLANIP, getLANIP, deleteLANIP + registration)
  rbac.go            # RBAC mutations (assignRole, removeRole, assignPermission, etc.)
  jwt.go             # JWT sign/verify
  oauth.go           # OAuth state + consumeState; GoogleProvider; MicrosoftProvider
  middleware.go      # HTTP middleware (cookie, JWT, bearer)
  migrate.go         # schema migration
  sql.go             # database/sql adapter
  api_token.go       # API token generation

user/ui/             # package userui ‚Äî frontend only, NO backend imports
  modules.go         # UIModules() + form registration
  login.go           # loginModule (struct + form + OnMount)
  register.go        # registerModule (struct + form + OnMount)
  profile.go         # profileModule (struct + form + OnMount)
  oauth.go           # oauthModule (struct + OnMount)
  lan.go             # lanModule (struct + OnMount)
  ssr/               # optional: SSR HTML rendering (if needed separately)
    login.go         # loginModule.RenderHTML()
    register.go      # registerModule.RenderHTML() + HandleRegister
    profile.go       # profileModule.RenderHTML() + HandleProfile
    oauth.go         # oauthModule.RenderHTML()
    lan.go           # lanModule.RenderHTML()
```

> **Note on SSR**: `RenderHTML()` methods are called by the backend (SSR). If kept in `ui/`,
> the backend would import `user/ui` (a frontend package). **Preferred**: put SSR rendering
> methods in `user/server/ssr.go` (or inline in `server/`) alongside their HTTP handlers,
> not in `ui/`. The `ui/` package contains only `OnMount()` and form wiring, which compile
> to WASM.

## go.mod impact

`user/server/` and `user/ui/` are **sub-packages of the same module** (`github.com/tinywasm/user`),
not separate modules. No new `go.mod` files. The existing `go.mod` at `user/` covers all
sub-packages.

Importers change:
- Was: `import "github.com/tinywasm/user"` ‚Üí `user.New(...)`, `user.UIModules()`
- Now:
  - Backend: `import "github.com/tinywasm/user/server"` ‚Üí `userserver.New(...)`
  - Frontend: `import "github.com/tinywasm/user/ui"` ‚Üí `userui.UIModules()`
  - Shared types: `import "github.com/tinywasm/user"` ‚Üí `user.Config`, `user.User`, etc.

All downstream packages (`app`, `goflare`, etc.) that import `user` must be updated to use
`user/server` or `user/ui` as appropriate.

## Constraints

- RULE: No `//go:build` tags anywhere in the restructured code. Package boundary = build boundary.
- RULE: `user/server` must NEVER import `user/ui`.
- RULE: `user/ui` must NEVER import `user/server`.
- RULE: Both `user/server` and `user/ui` may import root `user` for shared types.
- RULE: `models_orm.go` stays at root ‚Äî it is generated by `ormc` from `models.go` at root.
- RULE: `ormc` is run at the root (`user/`) only ‚Äî `server/` and `ui/` have no models.
- RULE: No standard library in WASM-compiled code (`user/ui/`). Use `github.com/tinywasm/fmt`.
- RULE: Do NOT copy `models_crud_back.go` ‚Äî it is deleted. `ReadAll*` is now generated by `ormc`.
